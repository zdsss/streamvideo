## 1. 抖音开放平台（推荐第一接入）

**平台地址**: https://open.douyin.com

### API 能力
- **视频上传**: `POST /api/media/upload/` — 分片上传视频文件
- **视频发布**: `POST /api/media/publish/` — 发布已上传的视频
- **发布状态查询**: `GET /api/media/publish/status/` — 查询审核状态

### 接入流程
1. 注册开发者账号（需企业资质或个人认证）
2. 创建应用，获取 client_key / client_secret
3. 用户 OAuth 2.0 授权（授权码模式）
4. 获取 access_token，调用 API

### OAuth 流程
```
用户点击授权 → 跳转抖音授权页 → 用户同意 → 回调带 code
→ 后端用 code 换 access_token → 存储 token → 调用 API
```

### 所需权限
- `video.create` — 创建视频
- `video.data` — 查询视频数据（可选）

### 限制
- 视频大小上限：128MB（分片上传可更大）
- 每日发布上限：取决于账号等级
- 审核时间：通常 1-24 小时
- 需要用户主动授权，不能代替用户操作

### 接入工作量
- OAuth 授权流程：~2天
- 视频上传+发布：~2天
- 状态轮询+错误处理：~1天

## 2. 快手开放平台（推荐第二接入）

**平台地址**: https://open.kuaishou.com

### API 能力
- **视频上传**: 分片上传接口
- **视频发布**: 发布接口（标题、描述、标签）
- **状态查询**: 查询审核和发布状态

### 接入流程
与抖音类似：注册开发者 → 创建应用 → OAuth 授权 → 调用 API

### 限制
- 需要企业开发者资质
- 审核周期较长
- API 文档质量一般

### 接入工作量
- 与抖音类似，约 5 天

## 3. B站（受限，第三优先）

**平台地址**: https://openhome.bilibili.com

### 现状
- B站有开放平台，视频投稿 API 存在但需要申请审核
- 社区有成熟的非官方工具：`biliup-rs`（Rust 实现，cookie 认证，支持分片上传）
- 社区 API 文档：`github.com/SocialSisterYi/bilibili-API-collect`

### 官方 API 流程
- OAuth 2.0 授权码模式
- 上传流程：获取上传地址 → 分片上传（5-10MB/片）→ 合并 → 提交投稿
- 需要实名认证 B站账号 + 应用审核

### 可行方案
- **方案 A**: 申请 B站开放平台（需审核，周期长）
- **方案 B**: 集成 `biliup-rs`（cookie 认证，QR 码登录，社区成熟方案）
- **方案 C**: 仅提供"一键复制信息 + 打开投稿页"的辅助功能

### 建议
优先方案 B（集成 biliup-rs），降级方案 C。

## 4. 小红书（暂不接入）

### 现状
- 小红书**没有公开的视频发布 API**
- 创作者平台不提供第三方自动化发布能力
- 所有发布必须通过小红书 App 或网页端手动操作
- 反自动化检测严格

### 建议
不接入。可提供"导出到本地 + 复制文案"的辅助功能。

---

## 5. 微信视频号（暂不接入）

### 现状
- 微信视频号**没有公开的视频发布 API**
- 视频号深度绑定微信生态，不对外开放发布能力
- 微信开放平台的能力主要在小程序、公众号，不包括视频号发布

### 建议
不接入。

## 实施建议

### 第一阶段：抖音分发（2-3周）
1. 注册抖音开放平台开发者
2. 实现 OAuth 授权流程（前端授权页 + 后端 token 管理）
3. 实现视频上传 + 发布
4. 实现状态轮询
5. 前端"发布到抖音"按钮

### 第二阶段：快手分发（2周）
复用抖音的架构，适配快手 API

### 第三阶段：B站辅助投稿（1周）
- "一键复制标题+描述+标签"
- "打开 B站投稿页"按钮

### 对 StreamVideo 的要求
- 需要一个公网可访问的回调 URL（OAuth redirect_uri）
- 需要 HTTPS（平台要求）
- 需要持久化存储 OAuth token（access_token + refresh_token）
- 建议部署到有公网 IP 的服务器

## 风险提示

1. **开发者资质**: 抖音/快手开放平台可能需要企业资质，个人开发者权限受限
2. **审核周期**: 应用审核可能需要 1-2 周
3. **Token 过期**: access_token 有效期通常 15 天，需要 refresh_token 机制
4. **内容审核**: 平台会对发布内容进行审核，可能被拒
5. **频率限制**: API 调用有频率限制，需要排队机制
