# StreamVideo V3.1 重构实施计划

## Context

StreamVideo 是一个多平台直播录制系统，正在从"录制工具"向"内容资产化引擎"演进。本次 V3.1 重构的目标是：

1. **精简冗余功能** — 删除已失效或低价值的功能（Stripchat、PWA、热力图等）
2. **优化核心体验** — 提升 FlashCut 模块的准确率和易用性（高光检测、字幕、封面、切片编辑）
3. **完善分发闭环** — 实现抖音/快手平台的视频发布能力
4. **商业化基础** — 用户系统和配额管理已完成，需补充支付集成

根据调研发现：
- **Phase 1（代码精简）和 Phase 2（核心优化）已完成**，代码中已无 Stripchat、PWA、热力图相关代码
- **Phase 3（分发闭环）和 Phase 4（商业化）已完成基础框架**，但抖音/快手 OAuth 和实际 API 对接尚未实现
- 项目已有完整的数据库表结构、API 端点、前端 UI 框架

## 实施计划

### Phase 3：分发闭环（预计 3-4 周）

#### Task 3.1：抖音 OAuth 授权流程（5 天）

**目标**：实现抖音开放平台 OAuth 2.0 授权码模式

**前置条件**：
- 需要注册抖音开放平台开发者账号
- 需要创建应用，获取 `client_key` 和 `client_secret`
- 需要配置回调 URL（需要 HTTPS + 公网可访问）

**实施步骤**：

1. **后端 OAuth 端点**（`server.py`）
   - `GET /api/oauth/douyin/authorize` — 生成授权 URL
   - `GET /api/oauth/douyin/callback` — 接收授权码，换取 token
   - `POST /api/oauth/douyin/refresh` — 刷新 token
   - `DELETE /api/oauth/douyin/revoke` — 撤销授权

2. **凭据存储**（`database.py`）
   - 新增表 `platform_credentials`
   - 字段：user_id, platform, access_token, refresh_token, expires_at, created_at

3. **前端授权 UI**（`static/index.html`）
   - 设置页面新增"平台授权"区块
   - 抖音授权按钮 → 弹出授权窗口
   - 显示授权状态和用户信息
   - 撤销授权按钮

4. **Token 管理**（`distribute.py`）
   - `get_credentials(user_id, platform)` 方法
   - 自动刷新过期 token

**验收标准**：
- 用户可通过前端完成抖音授权
- Token 正确存储到数据库
- Token 过期自动刷新

**关键文件**：
- `server.py` — OAuth 端点
- `database.py` — 凭据表
- `distribute.py` — Token 管理
- `static/index.html` — 授权 UI

---

#### Task 3.2：抖音视频发布实现（5 天）

**目标**：实现抖音视频上传和发布功能

**API 参考**：
- 上传：`POST /api/media/upload/` — 分片上传
- 发布：`POST /api/media/publish/` — 发布视频
- 状态查询：`GET /api/media/publish/status/`

**实施步骤**：

1. **DouyinPublisher 实现**（`distribute.py`）
   - `authenticate(credentials)` — 验证 token
   - `upload(task, credentials, progress_callback)` — 分片上传
   - `publish(media_id, task, credentials)` — 发布视频
   - `check_status(item_id, credentials)` — 查询审核状态

2. **分发任务执行**（`distribute.py`）
   - `execute_task(task_id)` 实现
   - 失败自动重试（最多 3 次）

3. **状态轮询**（`server.py`）
   - 后台任务：每 5 分钟轮询 `processing` 状态
   - WebSocket 推送状态变更

4. **错误处理**
   - Token 过期 → 自动刷新
   - 上传失败 → 重试
   - 审核失败 → 记录原因

**验收标准**：
- 可从 StreamVideo 发布切片到抖音
- 上传进度实时显示
- 审核状态自动更新
- 失败自动重试

**关键文件**：
- `distribute.py` — DouyinPublisher
- `server.py` — 状态轮询

### Phase 4：商业化基础（预计 2-3 周）

#### Task 4.1：支付集成（Stripe）（7 天）

**目标**：实现订阅支付和自动续费

**前置条件**：
- 注册 Stripe 账号
- 创建产品和价格（Free/Pro/Team）

**实施步骤**：

1. **Stripe SDK 集成**（新增 `payment.py`）
   - 安装 `stripe` Python SDK
   - 配置 API Key
   - 实现 `PaymentManager` 类

2. **订阅流程**（`payment.py`）
   - `create_checkout_session(user_id, tier)` — 创建支付会话
   - `handle_webhook(event)` — 处理 Stripe Webhook
   - `cancel_subscription(user_id)` — 取消订阅

3. **API 端点**（`server.py`）
   - `POST /api/payment/checkout` — 创建支付会话
   - `POST /api/payment/webhook` — Stripe Webhook
   - `POST /api/payment/cancel` — 取消订阅
   - `GET /api/payment/status` — 查询订阅状态

4. **前端支付 UI**（`static/index.html`）
   - 套餐选择页面（Free/Pro/Team 对比表）
   - 升级按钮 → 跳转 Stripe Checkout
   - 订阅管理页面

5. **数据库扩展**（`database.py`）
   - `users` 表新增字段：stripe_customer_id, stripe_subscription_id, subscription_status, subscription_expires_at

**验收标准**：
- 用户可通过 Stripe 升级套餐
- 订阅自动续费
- 取消订阅后降级到 Free

**关键文件**：
- `payment.py`（新增）
- `server.py`
- `database.py`
- `static/index.html`
