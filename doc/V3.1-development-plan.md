# StreamVideo V3.1 开发计划

> 创建时间：2026-04-15
> 状态：Phase 1 已完成

## 产品方向

StreamVideo 正从"录制工具"向"内容资产化引擎"演进。V3.1 目标：精简冗余功能，聚焦核心流程"录制 → 切片 → 分发"。

---

## Phase 1：代码精简（已完成）

### Task 1.1：删除 Stripchat 平台支持 ✅
- 移除 StripchatRecorder 类、URL 解析、平台注册
- 删除 record.sh、hls_proxy.py、monitor.sh
- 清理前端平台列表和文档

### Task 1.2：精简推送通道 ✅
- 只保留 Discord 和 Telegram

## Phase 2：核心体验优化（已完成）

### Task 2.1：AI 高光检测优化 ✅
- 扩展关键词库至 54 个（电商/游戏/通用，分三级权重）
- 新增礼物事件信号源（gift_spike）
- 弹幕密度动态阈值（短/中/长直播不同 σ 倍数）
- 时间窗口平滑（连续 2 窗口超阈值才算，防单点误判）
- 密度滑动窗口步进优化（5s 步进减少计算量）

### Task 2.2：Whisper 中文字幕优化 ✅
- 默认模型升级为 small（可配置到 large-v3）
- 中文初始提示词引导更好识别
- 标点恢复（句末自动加句号）
- 长句自动分割（每行不超过 20 字）
- 新增 VTT 格式导出
- 后处理管线：标点恢复 → 长句分割 → 空段过滤

### Task 2.3：封面生成质量提升 ✅
- 基于高光时刻选帧（highlight_time 参数）
- 多候选帧亮度评分（选最佳帧）
- 多尺寸输出（vertical/horizontal/square）
- generate_multi_size() 一次生成三种尺寸

### Task 2.4：切片预览与编辑 ✅
- 内嵌视频预览播放器（选中文件后加载）
- 双滑块时间范围选择器（可视化起止点）
- 预览按钮（播放选定范围，到结束点自动暂停）
- 水印文字输入（传递到后端）
- 精确数字输入 + 滑块联动

## Phase 3：分发闭环（已完成基础框架）

### Task 3.1：分发平台 API 调研 ✅
- 调研报告：`doc/platform-api-research.md`
- 抖音/快手有开放 API，可接入；B站半开放；小红书/微信视频号无 API
- 推荐优先级：抖音 > 快手 > B站辅助投稿

### Task 3.2：分发通用框架 + UI ✅
- `distribute.py`：DistributeManager + BasePlatformPublisher + MockPublisher
- DouyinPublisher / KuaishouPublisher 骨架（待 OAuth 配置后实现）
- 数据库 distribute_tasks 表
- API 端点：创建任务、执行、状态查询、任务列表
- 前端分发弹窗：平台选择、标题描述标签、状态追踪、轮询

---

## Phase 4：商业化基础（已完成）

### Task 4.1：用户系统 ✅
- `auth.py`：AuthManager（注册/登录/Session/注销）
- SHA-256 密码哈希 + 随机盐
- Session token（7天有效期）+ 过期清理
- API 端点：register、login、logout、me、users
- 数据库 users + user_sessions 表

### Task 4.2：配额系统完善 ✅
- 三级套餐定义（Free/Pro/Team）含功能差异化
  - Free: 3 clips/day, 720p, 水印, 3 主播
  - Pro: 50 clips/day, 1080p, 无水印, 20 主播, 云上传, H.265
  - Team: 无限, 1080p, 无水印, 无限主播, API 访问
- get_tier_info() 返回完整套餐信息
- check_feature() 功能权限检查
- get_usage_history() 使用历史
- API 端点：/api/tiers 套餐定义、/api/quota/{user}/history 使用历史
